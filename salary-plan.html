LTV基軸 統一給与制度

ベットランクツアーズ — 全部署を「顧客生涯価値」で連動させる新給与体系

① 設計思想 — なぜLTVなのか

会社の利益（LTV）に対してどれだけ貢献したかで全員を評価する。 売上だけ・新規だけ・部署バラバラでの評価をやめ、全員共通の指標に統一。
❌ NG — 旧型評価
  • 売上だけで評価
  • 新規取ったら終わり
  • 部署ごとにバラバラ
  • 会社が赤字でも歩合は支給
✅ 正解 — LTV基軸評価
  • 顧客が最終的にいくら落としたかで評価
  • 新規→現地→継続を連動
  • 全部署が同じKPIを追う
  • LTVが伸びれば全員が利益を共有

🟢 テレベット / アバター

追うKPI

  • 継続率（担当顧客の月次継続率）
  • 売上（担当顧客のテレベ月次プレイ額）

インセンティブ

項目報酬
売上歩合5〜10%
3ヶ月継続ボーナス（同顧客）+¥20,000
6ヶ月継続ボーナス（同顧客）+¥50,000

→ 継続させるほど・LTVを伸ばすほど指数関数的に報酬が伸びる。短期売上だけでなく長期関係を評価。

⑦ デポジット連動ボーナス（全部署対象）

🌟 デポジットは最重要KPI

理由:

  • 顧客の本気度が高い → 離脱しない
  • LTVが伸びる確率が圧倒的に高い
  • 会社のキャッシュフローが安定

分配型インセンティブ — 部署連携を強化

部署役割報酬
CSマーケアポ獲得（デポ説明アポ）¥10,000 / 件
現地スタッフデポジット契約成立¥20,000 / 件
テレベット/アバターデポジット回収（実プレイ）¥10,000 / 件

→ 1人の顧客のデポジット成立で 3部署が同時に報酬を得る構造。 部署をまたいだ連携が報酬で促進される。

⑧ 旧設計（A/Bプラン）の問題点と修正

旧設計の課題新設計での解決
「会社がマイナスにならない」が抽象的 → LTVとCPAで数値化。広告費上限・歩合率を客観的に判断可能
「デポジット重要」とは言っているが評価に落ちていない → デポジットを固定報酬KPI化（¥10K〜¥20K/件、3部署分配）
部署ごとに評価軸がバラバラ（CSは件数、現地はランク、テレベは%） → 全部署LTVに紐付け。共通言語で評価
新規取って終わり、関係維持が評価されない → 継続ボーナス・LTV貢献ボーナスで長期関係を評価

⑨ 運用ルール

支給タイミング

  • 給与支給日: 毎月20日（前月分を支給）
  • LTV基本歩合は仮算定 → 12ヶ月後に確定額で精算
  • デポジット・継続ボーナスは月次確定支給

会社黒字維持の安全装置

  • 全インセンティブの合計 ≤ 月次粗利の25%（超過時は按分減額）
  • 会社月次純利益赤字時はLTVボーナス支給率を50%に減額
  • 基本給は会社収支に関わらず100%保証

LTV確定ルール

  • 顧客との関係終了（明確な離脱）時 = 確定
  • 最終来店から12ヶ月経過 = 確定
  • 担当変更時 = 引き継ぎ時点で旧担当のLTV確定

⑩ 設計まとめ

  • 広告 → LTVで判断（CPA > LTV×40%は警告）
  • インセン → LTVベース統一（部署で計算式は違うが指標は同じ）
  • デポジット → 強化KPI（3部署同時支給で連携促進）
  • 部署 → 分断せず連動（新規→現地→継続を全員で追う）
  • 会社黒字維持（粗利25%上限・赤字時自動減額）
